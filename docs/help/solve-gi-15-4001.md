# 解决原神 15-4001 错误

_关于原神检测 d3dxSkinManage 进程导致不能游玩的解决办法。_

## 1. 前引

![](/static/image/67aa8b1c.png)

原神在 5.8 版本更新后加强了对第三方工具进程的检测，其中包括 __d3dxSkinManage__、__胡桃工具箱__ 及 __XXMI__ 等。一旦检测到这些进程，游戏会强制终止当前会话并返回错误代码 `15-4001`，导致玩家被踢出游戏。

目前，该检测机制仅依据进程名称进行判断。例如，只要进程名为 `d3dxSkinManage.exe`，即使该程序仅为简单的 `Hello World` 应用，也会触发拦截。因此，通过修改相关进程的名称即可有效规避该项检测。

此外，原神并未将检测逻辑完全内置在客户端中，而是会将扫描到的进程信息上传至服务器进行验证。这意味着，阻止游戏客户端向服务器发送相关数据，同样可以避免触发 `15-4001` 错误。

需说明的是，本文仅提供思路与分析方法，**不涉及任何具体工具的分发或推荐**，仅介绍一种**简单**、**安全**且**可行**的规避策略。

## 2. 解决方法

我们将使用 [火绒](https://www.huorong.cn) 作为示例，其他安全软件也类似。

![](/static/image/38d457a8.png)
在 __火绒__ 的 __访问控制__ 中找到 __IP 协议控制__。

![](/static/image/b00b1217.png)
开启 __IP 协议控制__ 并添加一条规则；

![](/static/image/fa129e37.png)
__应用程序__ 设置为 原神游戏文件路径；  
__操作__ 设置为 __阻止__；  
__方向__ 设置为 __所有__；  
__协议__ 设置为 __TCP__；

::: tip 提示
__应用程序__ 可以简短的设置为 `*\YuanShen.exe` 但不能是 `YuanShen.exe` 这与火绒的匹配规则有关。
:::

![](/static/image/d8820080.png)
创建完成后先**关闭**这条规则，否则会**卡在**检查更新阶段，在看到大门立起来之后再**打开**这条规则，届时即可以进入游戏了。

原神的游戏数据传输采用 UDP 协议，而进程检测信息的反馈则依赖于 TCP 连接。因此，仅阻止所有 TCP 连接并不会影响正常的游戏体验，但可有效防止游戏客户端将检测到的第三方进程信息上传至服务器，从而避免触发 `15-4001` 错误。
